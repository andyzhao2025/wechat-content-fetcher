澳洲消防 ASE 设计规范和准入要求

Foring.Wu

在澳洲消防领域，ASE 和 OWS 其实是两种完全不同的系统，但它们都跟消防主机（Fire Indicator Panel / FIP）有密切接口关系。

澳洲的消防主机（尤其是主流品牌如 Notifier、Hochiki、Tyco、Ampac、Brooks、FireFinder 等）设计时通常都会预留专用接口/位置/扩展槽来安装或连接 ASE 和 OWS，就是你看到的“留个位置”现象。下面分别解释清楚：

1. ASE（Alarm Signalling Equipment）——报警信号传输设备

  • 这是谁提供的？
    不是普通消防厂家（fire panel制造商）自己做的。ASE 是由消防局（Fire Brigade）或其指定/授权的监控服务提供商统一提供/拥有/维护的专用设备。

  • 不同州有不同主流品牌：

    • 维州（VIC）：Romteck、Chubb Code Red、Centaur 等

    • 昆士兰（QLD）：FireNet 体系的 ASE

    • 西澳（WA）：DFES 的 Code Red（Chubb 拥有）

    • 新州（NSW）、ACT 等：Romteck Grid 或其他授权品牌

  • 澳洲标准 AS 1670.3 强制要求某些建筑（大型商业、公共、高层等）必须把火警信号直接传给消防局，所以 ASE 是强制性监控接口。

  • 安装位置：
    通常是一个红色小盒子（red box），直接装在消防主机柜里面或旁边，通过干接点（relay output）、RS232/485 等接口跟主机连接。
    所以澳洲主机设计时必须预留空间、继电器输出或串口给 ASE 使用（很多面板在出厂就标明“ASE Interface”或预留扩展位）。

  • 产权和维护：
    ASE 设备产权属于消防局/监控公司，安装后归他们所有，业主/用户要交年费（monitoring fee），消防公司或监控服务商负责升级、维修（比如 3G 转 4G/5G 升级就是统一强制做的）。

2. OWS（Occupant Warning System）——建筑内人员警报系统

  • 这是谁提供的？
    就是普通的消防设备厂家做的，属于消防主机系统的一部分或扩展。

  • 很多澳洲消防主机品牌直接内置 OWS 模块（比如 Brooks FT系列内置 20W OWS、Ampac、Notifier 等都有 OWS 选项）

  • 或者通过扩展板/独立 EWIS（Emergency Warning & Intercommunication System）柜连接

  • 功能：发出警铃声（Alert tone → Evac tone）、语音广播、连接喇叭/闪光灯等，目的是让楼里的人听到/看到要疏散。

  • 安装位置：
    通常是主机内置放大器+扬声器接口，或旁边加 EWIS 柜。澳洲标准 AS 1670.1 和 NCC（Building Code）要求大部分多层/多人建筑必须有 OWS，所以主机设计时必须预留功率输出、语音模块、麦克风接口等位置。

为什么澳洲主机都要“留位置”给 ASE 和 OWS？

  • 法规强制：澳洲的 Building Code of Australia (NCC) + AS 1670 系列标准，对不同类型建筑有严格的**监控（ASE）和人员警报（OWS）**要求。

  • 标准化设计：主机制造商为了全国通用，从设计阶段就预留好接口（relay、serial port、expansion slot、power output），方便后期插 ASE 盒子或加 OWS 模块。

  • 结果：你看到的澳洲消防主机（尤其是商用/工业级）外观上总有明显扩展空间、预留线槽、标着“ASE”或“EWIS/OWS”的接口，就是为了满足这个强制要求。

总结一句话：

  • ASE → 消防局/监控公司提供的“报警外传”专用盒子（红色的），主机留位置给它接。

  • OWS → 消防厂家自己做的“楼内喊话/警铃”系统，主机通常内置或直接扩展。

两者都是澳洲消防强制配套，所以主机不留位置就卖不出去、也通不过消防验收。😄
